# PRD — حاسبة التسعير للأسر المنتجة

## ١. نظرة عامة
تطبيق ويب باللغة العربية يساعد الأسر المنتجة والأفراد في السعودية على تسعير منتجاتهم الغذائية بدقة. يحسب التكلفة الفعلية (مواد + طاقة + وقت) ويقترح سعر بيع بناءً على هامش ربح، مع شرح تعليمي شفاف يوضح للمستخدم من أين جاء كل ريال.

## ٢. المشكلة
أغلب الأسر المنتجة تسعّر بالحدس: إما تخسر بدون ما تدري، أو تبالغ في السعر فما يشتري منها أحد. ما عندها أداة بسيطة بالعربي تحسب لها التكلفة الحقيقية وتعلّمها كيف تفكر في التسعير.

## ٣. الجمهور المستهدف
- أسر منتجة في السعودية تبيع أكل منزلي (كيك، معجنات، طبخات)
- أفراد بدون أي خبرة محاسبية أو تقنية
- المستخدم النموذجي: أم تستخدم الجوال بشكل أساسي، تبيع عبر واتساب وانستقرام

## ٤. النطاق — النسخة الأولى (MVP)
**فقط منتجات الأكل.** باقي الفئات (خياطة، إكسسوارات، خدمات) تأتي لاحقاً.

## ٥. المتطلبات الوظيفية

### ٥.١ إدارة المكونات
المستخدم يضيف مكون بـ:
- اسم المكون (طحين، سكر، زبدة، ...)
- وزن أو حجم العبوة المشتراة (جرام / مل)
- سعر العبوة بالريال
- تاريخ آخر تحديث للسعر (تلقائي)

المستخدم يقدر يعدل أو يحذف أي مكون. المكونات تظهر في قائمة قابلة للبحث.

### ٥.٢ إدارة الوصفات
المستخدم ينشئ وصفة فيها:
- اسم الوصفة
- عدد الحصص التي تنتجها (مثل: ١٢ قطعة)
- قائمة المكونات + الكمية المستخدمة من كل واحد (جرام/مل)
- وقت التحضير (دقائق)
- وقت الطبخ/الفرن (دقائق)
- مصدر الطاقة: كهرباء / غاز / كلاهما

الوصفات تُحفظ بشكل دائم، ويقدر المستخدم يرجع لها يعدلها أو يحذفها.

### ٥.٣ حساب التكلفة
لكل وصفة، البرنامج يحسب:

**أ. تكلفة المواد:**
لكل مكون = (سعر العبوة ÷ وزن العبوة) × الكمية المستخدمة

**ب. تكلفة الطاقة:**
- الكهرباء: متوسط ١٨ هللة لكل كيلوواط ساعة (قابل للتعديل في الإعدادات حسب شريحة المستخدم)
- استهلاك الفرن الكهربائي التقريبي: ٢ كيلوواط في الساعة
- الغاز: حساب تقريبي بناءً على متوسط سعر أسطوانة الغاز السعودية ومدة استخدام الموقد/الفرن

**ج. تكلفة الماء:** تقديرية بسيطة (يمكن تجاهلها في النسخة الأولى لو معقدة)

**د. قيمة الوقت:**
المستخدم يحدد قيمة ساعة عمله في الإعدادات (افتراضي: ٢٥ ريال/ساعة).
قيمة الوقت للوصفة = (وقت التحضير + وقت الطبخ) × السعر الساعي.

**هـ. إجمالي التكلفة الفعلية = أ + ب + ج + د**

### ٥.٤ اقتراح السعر
البرنامج يعرض عدة خيارات هامش ربح:
- ربح ٢٠٪
- ربح ٣٠٪
- ربح ٥٠٪
- ربح مخصص (المستخدم يدخل النسبة)

لكل خيار، يُعرض:
- السعر المقترح للوصفة كاملة
- السعر المقترح للحصة الواحدة (مثل: للقطعة)

### ٥.٥ الشفافية التعليمية ⭐ (الميزة المحورية)
بعد الحساب، يعرض البرنامج تفصيل واضح بهذا الشكل:

> **كيكة الشوكولاتة كلفتك فعلياً ١٨.٥٠ ريال:**
> - ١٢.٣٠ ريال — مواد خام
> - ٢.٢٠ ريال — غاز الفرن (٤٠ دقيقة)
> - ٤.٠٠ ريال — قيمة وقتك (٤٥ دقيقة)
>
> **السعر المقترح بربح ٣٠٪: ٢٤ ريال للكيكة كاملة، أو ٢ ريال للقطعة.**

الهدف: المستخدم يفهم **من وين جا الرقم**، مو بس يشوفه. هذي الميزة هي اللي تفرق البرنامج عن أي حاسبة عادية.

### ٥.٦ الإعدادات
المستخدم يقدر يعدل:
- قيمة ساعة عمله
- تعرفة الكهرباء (هللة/كيلوواط ساعة)
- سعر أسطوانة الغاز

### ٥.٧ تسجيل الدخول
- تسجيل بإيميل وكلمة مرور عبر Supabase Auth
- كل مستخدم يشوف مكوناته ووصفاته فقط (Row Level Security)

## ٦. المتطلبات غير الوظيفية

### اللغة والاتجاه
- **عربي بالكامل** — اللغة الوحيدة في النسخة الأولى
- **RTL صحيح ١٠٠٪** في كل عناصر الواجهة (`dir="rtl"` على html + استخدام logical properties في CSS)
- خط عربي واضح: Cairo أو IBM Plex Sans Arabic أو Tajawal من Google Fonts

### التصميم
- **Mobile-first** — الجمهور كله على الجوال
- بسيط جداً، يستهدف مستخدمات غير تقنيات
- ألوان هادئة، أزرار كبيرة، نصوص واضحة بحجم مقروء
- responsive يشتغل من شاشة ٣٢٠px إلى الديسكتوب
- ما يستخدم مصطلحات تقنية في الواجهة

### الأداء
- يفتح بسرعة على شبكات الجوال (3G/4G)
- بدون lag حتى على أجهزة متوسطة

### إمكانية الوصول
- contrast عالي بين النص والخلفية
- أحجام لمس مناسبة (٤٤px على الأقل للأزرار)

## ٧. التقنيات
- **Frontend:** HTML + CSS + JavaScript (vanilla في البداية، بدون frameworks ثقيلة)
- **Backend + Database:** Supabase (PostgreSQL + Auth — الخطة المجانية)
- **الاستضافة:** Vercel أو Netlify (مجاني)
- **التحكم بالإصدارات:** Git + GitHub
- **العملة:** الريال السعودي (SAR) فقط

## ٨. خارج نطاق النسخة الأولى
- أي منتجات غير الأكل
- جلب أسعار المواد من السوق تلقائياً (scraping/APIs)
- تطبيق جوال أصلي iOS/Android
- أي لغة غير العربية
- مشاركة الوصفات بين المستخدمين
- مدفوعات أو اشتراكات
- تقارير وإحصائيات متقدمة
- طباعة فواتير أو ملصقات
- إشعارات

## ٩. معايير النجاح
- المستخدم يضيف مكون جديد في أقل من ٣٠ ثانية
- المستخدم ينشئ وصفة كاملة ويحصل على السعر في أقل من دقيقتين
- التفصيل التعليمي مفهوم لشخص بدون أي خلفية محاسبية
- الواجهة العربية صحيحة ١٠٠٪ بدون أي عنصر مكسور أو معكوس

## ١٠. خطة المراحل
**المرحلة ١:** الواجهة + إدارة المكونات والوصفات بدون قاعدة بيانات (تخزين محلي مؤقت)
**المرحلة ٢:** ربط Supabase + تسجيل الدخول + الحفظ الدائم
**المرحلة ٣:** تحسين التصميم + الشفافية التعليمية بشكلها النهائي
**المرحلة ٤:** النشر على Vercel + اختبار مع مستخدمين حقيقيين
